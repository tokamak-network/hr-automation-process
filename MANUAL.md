# 📖 Tokamak HR Automation — 사용 매뉴얼

> 이 문서는 비개발자도 이해할 수 있도록 작성되었습니다.
> 기술적 내용은 최소화하고, "무엇을 할 수 있고, 어떻게 쓰는지"에 집중합니다.

---

## 🤔 이 시스템은 뭐하는 거야?

**한 줄 요약:** 채용 후보자의 GitHub 결과물을 AI가 자동으로 분석해서 점수를 매겨주는 시스템.

크게 두 가지 기능이 있어:

### 기능 1: 후보자 평가 (Core Pipeline)
누군가 "토카막에서 일하고 싶어요"라고 왔을 때:
1. 그 사람이 만든 GitHub 레포(코드 저장소) 주소를 시스템에 등록
2. AI가 그 레포를 자동으로 분석
3. 5가지 기준으로 점수를 매기고 "채용 추천/비추천" 판단

### 기능 2: 후보자 자동 발굴 (Monitor)
우리가 찾아가지 않아도:
1. 토카막 네트워크 GitHub에 관심을 보이는 외부 사람들을 자동으로 감지
2. 예: 우리 레포에 ⭐를 누르거나, 코드를 복사(Fork)하거나, 의견을 남긴 사람들
3. 그 사람들의 프로필을 AI가 분석해서 "이 사람 괜찮을 수도 있어요"라고 알려줌

---

## 🖥️ 화면 구성

시스템을 켜면 웹 브라우저에서 4개 페이지를 볼 수 있어:

### 1. Candidates (후보자 목록) — 메인 페이지
- 등록된 모든 후보자가 표로 나옴
- 이름, 이메일, 점수, 추천 등급, 상태가 한눈에 보임
- 상태: `submitted` (등록됨) → `analyzed` (분석 완료)

### 2. Submit (후보자 등록)
- 새로운 후보자를 등록하는 폼
- 입력할 것:
  - **Name**: 후보자 이름
  - **Email**: 후보자 이메일
  - **Repository URL**: 후보자가 만든 GitHub 레포 주소 (예: `https://github.com/홍길동/my-project`)
  - **Description**: 이 사람이 뭘 만들었는지 간단 설명 (선택)

### 3. Candidate Detail (후보자 상세)
- 후보자 목록에서 한 사람을 클릭하면 나오는 페이지
- AI가 분석한 상세 스코어카드:
  - 5가지 점수 (각 10점 만점)
  - AI가 작성한 평가 리포트 (한글/영문)
  - 최종 추천: Strong Hire / Hire / Maybe / Pass

### 4. Monitor (자동 발굴)
- "Scan" 버튼을 누르면 토카막 GitHub를 스캔
- 우리 팀원이 아닌 외부 활동자가 리스트로 나옴
- 각 사람의 GitHub 프로필 요약 + AI 점수

## 📋 실제 사용 시나리오

### 시나리오 A: 누군가 지원했을 때

```
1. 지원자가 이메일/LinkedIn DM으로 연락
   → "저 토카막에 관심 있습니다. 제가 만든 프로젝트입니다: github.com/xxx/yyy"

2. Jaden 또는 Irene이 시스템에 접속
   → http://localhost:3002 (또는 배포된 주소)

3. "Submit" 페이지로 이동
   → 이름, 이메일, 레포 URL 입력 → 제출

4. "Candidates" 목록에서 방금 등록한 사람 클릭

5. "Analyze" 버튼 클릭
   → AI가 레포를 다운로드하고 분석 시작 (1~2분 소요)

6. 분석 완료되면 스코어카드 확인
   → 점수가 높고 "Strong Hire"면 인터뷰 진행
   → "Pass"면 정중히 거절
```

### 시나리오 B: 유망한 사람을 먼저 찾고 싶을 때

```
1. "Monitor" 페이지로 이동

2. "Scan" 버튼 클릭
   → 토카막 GitHub의 모든 레포를 뒤져서 외부 활동자를 찾음
   → 우리 팀원 16명은 자동으로 제외됨

3. 결과 리스트 확인
   → "이 사람은 우리 레포에 Fork도 하고 PR도 보냈네?"
   → AI 점수가 높은 사람 위주로 확인

4. 유망한 사람에게 직접 연락
   → GitHub 프로필에서 이메일/SNS 확인
   → "토카막에 관심 있으신 것 같은데, Track B 제출해보실 의향 있으세요?"
```

### 시나리오 C: 케빈에게 후보자 보고

```
1. Candidates 목록에서 상위 후보자 확인
2. 각 후보자의 스코어카드 + AI 리포트를 스크린샷 또는 공유
3. "이 사람은 Technical 9점, Ecosystem Fit 8점이고 Strong Hire 입니다"
4. 케빈이 직접 레포를 보고 최종 판단
```

## 📊 AI 평가 기준 상세

AI가 후보자의 레포를 분석할 때 보는 5가지:

| 점수 | 무슨 뜻? | 예시 |
|------|----------|------|
| **Technical Completeness** (기술 완성도) | 코드가 잘 짜여져 있는지, 실행 가능한지 | "README도 있고, 테스트도 있고, 실제로 돌아감" → 높은 점수 |
| **Ecosystem Fit** (생태계 적합성) | 토카막의 기존 프로젝트와 시너지가 있는지 | "우리 L2 브릿지와 연동 가능한 프로젝트" → 높은 점수 |
| **Tokenomics Impact** (토크노믹스 영향도) | TON/STON 활용이나 프로토콜 수준 기여 가능성 | "시뇨리지 메커니즘을 활용한 새 DApp" → 매우 높은 점수 |
| **Innovation** (혁신성) | 새로운 접근인지, 차별점이 있는지 | "기존에 없던 ZK 프라이버시 풀 구현" → 높은 점수 |
| **AI Proficiency** (AI 활용도) | AI 도구를 잘 활용했는지 | "커밋에 AI 코파일럿 흔적, 문서 자동 생성 등" → 높은 점수 |

**최종 추천 등급:**
- 🟢 **Strong Hire** — 바로 인터뷰 잡아야 함
- 🔵 **Hire** — 충분히 괜찮음, 인터뷰 추천
- 🟡 **Maybe** — 애매함, 추가 확인 필요
- 🔴 **Pass** — 기준 미달

## ⚙️ 시스템 켜는 법 (처음 한 번만 설정)

### 사전 준비물
1. **GitHub Token** — GitHub 개인 설정에서 발급 (레포 읽기 권한)
2. **AI API Key** — OpenAI 또는 호환 API 키
3. **컴퓨터에 Python, Node.js 설치** (이건 이미 되어 있음)

### 설정 파일 만들기
`backend/` 폴더에 `.env` 파일을 만들고:
```
GITHUB_TOKEN=여기에_깃헙_토큰
AI_API_KEY=여기에_AI_키
AI_API_URL=https://api.openai.com/v1/chat/completions
AI_MODEL=gpt-4o-mini
```

### 서버 실행
```bash
# 터미널 1: 백엔드
cd backend
source venv/bin/activate
uvicorn main:app --port 8001

# 터미널 2: 프론트엔드
cd frontend
npm run dev -- -p 3002
```

### 접속
- 브라우저에서 http://localhost:3002 열기

## ❓ 자주 하는 질문

**Q: 지원자가 직접 이 시스템에 접속해서 등록하나요?**
A: 아니요. 현재는 내부 관리자(Jaden, Irene)만 사용합니다. 지원자는 이메일이나 LinkedIn으로 레포 URL을 보내고, 관리자가 시스템에 등록합니다. (향후 지원자 셀프 서비스 페이지 추가 예정)

**Q: 분석하는데 얼마나 걸려요?**
A: 레포 크기에 따라 1~3분 정도. AI가 코드를 읽고 평가 리포트를 작성하는 시간입니다.

**Q: AI 점수를 100% 신뢰해도 되나요?**
A: 참고 자료로 활용하세요. 최종 판단은 사람(케빈)이 합니다. AI는 1차 필터링 + 정리 도구입니다.

**Q: Monitor 스캔은 얼마나 자주 해야 하나요?**
A: 주 1~2회 정도면 충분합니다. 향후 자동 정기 스캔(cron) 기능 추가 예정.

**Q: 우리 팀원이 실수로 후보자로 잡히진 않나요?**
A: 내부 팀원 16명의 GitHub ID가 시스템에 등록되어 있어서 자동으로 제외됩니다.
