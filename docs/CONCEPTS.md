# Tokamak Network HR 솔루션 — 5개 컨셉 제안

> 배경: ~10명 규모, 이더리움 L2, USDT 월급여 + TOKAMAK 토큰 분기 인센티브, 독립계약자 형태, 한국 근로소득세 시뮬레이션, 기존 Hiring System(Next.js+FastAPI+SQLite) 통합

---

## 컨셉 A: Hiring 시스템 확장형 — "TokaHR Extend"

> **핵심 철학**: 이미 검증된 Hiring 시스템 위에 급여·세금·인사 모듈을 자연스럽게 얹는다.

### UI/UX 구조

**사이드바 메뉴:**
- 🏠 대시보드
- 👥 팀원 관리
- 💰 급여 관리
- 📊 세금 시뮬레이션
- 🎯 채용 (기존 Hiring)
- ⚙️ 설정

**네비게이션 흐름:**
기존 Hiring 시스템의 사이드바에 급여·팀원·세금 메뉴가 추가되는 형태. 기존 사용자가 새로운 네비게이션을 학습할 필요 없음. 채용 파이프라인에서 "합류" 시 자동으로 팀원 관리로 이관.

**주요 화면:**
- **대시보드**: 팀원 현황, 이번 달 급여 요약, 진행 중 채용 건수
- **팀원 관리**: 프로필, 계약 정보, 급여 이력, 지갑 주소
- **급여 관리**: 월별 USDT 지급 내역, 분기 인센티브, 지급 승인
- **세금 시뮬레이션**: 개인별 연간 소득 추정, 세액 시뮬레이션 결과, 적립금 현황

### 기능 목록

| 영역 | 기능 |
|------|------|
| 급여 | USDT 월급여 기록, TOKAMAK 분기 인센티브 기록, 지급일 환율/시세 자동 수집 |
| 세금 | 근로소득세 시뮬레이션, 월별·분기별 세액 추정, 적립금 관리 |
| 채용 | 기존 Hiring 그대로 (AI Track B 평가, GitHub Monitor, LinkedIn 소싱) |
| 팀원 | 프로필, 계약 관리, 지갑 주소, 급여 이력 |
| 근태 | 간단한 출근/퇴근 기록 (자율 보고) |
| 퇴사 | 퇴사 처리, 최종 정산, 적립금 정리 |
| 대시보드 | 팀 현황 요약 |

### 급여/소득세 계산 로직

**1. 월급여 (USDT → KRW)**
- 지급일 기준 USDT/KRW 환율 수집 (업비트 또는 외환 API)
- `월급여_KRW = USDT_금액 × 지급일_USDT_KRW_환율`

**2. 분기 인센티브 (TOKAMAK → KRW)**
- 지급일 기준 업비트 TOKAMAK/KRW 종가 수집
- `분기_인센티브_KRW = TOKAMAK_수량 × 지급일_업비트_종가`

**3. 연간 근로소득 추정**
- `연간_총소득 = Σ(월급여_KRW) + Σ(분기_인센티브_KRW)`
- 근로소득공제 적용 → 과세표준 산출
- 2024 근로소득세율표 적용:

| 과세표준 | 세율 | 누진공제 |
|----------|------|----------|
| ~1,400만 | 6% | 0 |
| ~5,000만 | 15% | 126만 |
| ~8,800만 | 24% | 576만 |
| ~1.5억 | 35% | 1,544만 |
| ~3억 | 38% | 1,994만 |
| ~5억 | 40% | 2,594만 |
| ~10억 | 42% | 3,594만 |
| 10억 초과 | 45% | 6,594만 |

**4. 월별/분기별 세액 시뮬레이션**
- 현재까지 누적 소득 기반으로 연말 예상 총소득 추정
- 예상 세액 산출 → 이미 시뮬레이션된 세액과 비교
- 월별 "예상 세부담" 그래프 제공

**5. 적립금(과세분 토큰) 관리**
- 시뮬레이션된 세액에 해당하는 TOKAMAK 수량 산출
- `적립_TOKAMAK = 세액_KRW ÷ 지급일_업비트_종가`
- 회사 보유 적립금 잔고 관리 (누적/인출/조정 이력)
- 연말 정산 후 과부족 정리

### 기술 스택

| 계층 | 기술 |
|------|------|
| Frontend | Next.js (기존 유지) |
| Backend | FastAPI (기존 유지) |
| DB | SQLite → PostgreSQL 마이그레이션 권장 |
| 외부 API | 업비트 API (TOKAMAK 종가), 환율 API (USDT/KRW) |
| 인증 | 기존 인증 체계 유지 |
| 배포 | 기존 인프라 유지 |

### 차별점

- **최소 개발 비용**: 기존 코드베이스에 모듈만 추가하므로 가장 빠르게 구축 가능
- **채용→인사 이관 자동화**: Hiring 파이프라인의 합격자가 곧바로 팀원 관리로 연결
- **학습 비용 제로**: 기존 사용자에게 익숙한 UI 그대로

## 컨셉 B: 미니멀 경량형 — "TokaPay"

> **핵심 철학**: 급여 지급과 세금 시뮬레이션, 딱 이 두 가지만 완벽하게.

### UI/UX 구조

**단일 페이지 + 탭 구조 (탑바):**
- 💰 급여 | 📊 세금 | 👤 팀원 | ⚙️ 설정

**네비게이션 흐름:**
SPA 느낌의 단일 화면. 탭 전환으로 급여 → 세금 → 팀원 이동. 모든 핵심 정보가 한 화면에 보이는 것을 지향. 채용은 기존 Hiring 시스템 링크로 연결 (외부 이동).

**주요 화면:**
- **급여 탭**: 이번 달 급여 리스트 (전체 팀원 한 눈에), 1클릭 기록, USDT/TOKAMAK 입력 시 실시간 KRW 환산 표시
- **세금 탭**: 팀원 선택 → 연간 소득 추정 → 세액 시뮬레이션 결과 즉시 표시
- **팀원 탭**: 이름, 월급여, 지갑주소, 계약 시작일 — 간단한 카드 리스트

### 기능 목록

| 영역 | 기능 |
|------|------|
| 급여 | USDT 월급여 기록, TOKAMAK 분기 인센티브 기록, 실시간 KRW 환산 |
| 세금 | 근로소득세 시뮬레이션, 적립금 현황 |
| 채용 | 기존 Hiring 시스템 외부 링크 |
| 팀원 | 기본 프로필 (이름, 급여, 지갑) |
| 근태 | ❌ (미포함 — 독립계약자이므로 불필요 판단) |
| 퇴사 | 간단한 비활성화 처리 |
| 대시보드 | ❌ (급여 탭이 곧 대시보드) |

### 급여/소득세 계산 로직

A 컨셉과 동일한 계산 로직 적용. 차이점:

- **실시간 환산**: 급여 입력 시 현재 시세 기반 KRW 즉시 표시 (확정은 지급일)
- **간편 시뮬레이터**: 슬라이더로 월급여·인센티브 조정 → 예상 세액 실시간 변동
- **적립금**: 단순 누적 테이블로 관리 (회사 보유 TOKAMAK 잔고)
- 복잡한 분석 없이 "이번 달 세금 얼마, 적립금 얼마" 핵심 수치만 표시

### 기술 스택

| 계층 | 기술 |
|------|------|
| Frontend | React (Vite) 또는 Svelte — 경량 SPA |
| Backend | Hono 또는 Express (단일 파일 수준) |
| DB | SQLite (파일 기반, 서버리스) |
| 외부 API | 업비트 API, 환율 API |
| 배포 | Cloudflare Workers 또는 Vercel |
| 인증 | 단순 비밀번호 또는 OAuth |

### 차별점

- **극단적 단순함**: 화면 3개, 기능 3개. 학습 비용 0
- **가장 빠른 개발**: 1~2주 내 MVP 가능
- **유지보수 최소**: 코드베이스 작음, 의존성 적음
- **Hiring은 별도 유지**: 급여 시스템과 채용 시스템의 관심사를 명확히 분리

## 컨셉 C: 대시보드/분석 중심형 — "TokaSight"

> **핵심 철학**: 데이터를 보면 의사결정이 보인다 — 인건비·세금·토큰의 모든 흐름을 시각화.

### UI/UX 구조

**사이드바 메뉴:**
- 📊 인사이트 대시보드
- 💰 급여 분석
- 🪙 토큰 분석
- 📈 세금 시뮬레이터
- 👥 팀원
- 🎯 채용 현황
- ⚙️ 설정

**네비게이션 흐름:**
대시보드가 랜딩. 차트와 KPI 카드가 중심. 각 섹션은 "요약 → 상세" 드릴다운 구조. 대시보드에서 이상치 클릭 → 해당 상세 화면으로 이동.

**주요 화면:**
- **인사이트 대시보드**: 월 총 인건비 추이, 토큰 가격 변동과 인건비 상관관계, 팀원별 급여 비중, 예상 세부담 게이지
- **급여 분석**: 월별/분기별 급여 지출 차트, USDT 환율 변동 영향 분석, 팀원별 비교
- **토큰 분석**: TOKAMAK 가격 추이 (업비트), 인센티브 지급 이력 vs 토큰 가격, 적립금 가치 변동
- **세금 시뮬레이터**: What-if 분석 (급여 인상 시 세금 변화, 토큰 가격 시나리오별 세부담)

### 기능 목록

| 영역 | 기능 |
|------|------|
| 급여 | 급여 기록 + 다차원 분석 (시계열, 환율 영향, 팀원별) |
| 세금 | 시뮬레이션 + What-if 시나리오 분석 + 세부담 예측 |
| 채용 | Hiring 시스템 데이터 연동 → 채용 퍼널 시각화 |
| 팀원 | 프로필 + 개인별 급여/세금 대시보드 |
| 근태 | 기본 기록 + 근무일수 통계 |
| 퇴사 | 퇴사 처리 + 비용 영향 분석 |
| 대시보드 | 핵심 KPI, 인건비 트렌드, 토큰 가격 연동 차트 |

### 급여/소득세 계산 로직

A 컨셉과 동일한 기본 로직 + 분석 확장:

- **환율 영향 분석**: USDT/KRW 환율 ±5% 변동 시 인건비 영향 시뮬레이션
- **토큰 가격 시나리오**: TOKAMAK 가격 Bull/Base/Bear 케이스별 세부담 비교
- **연간 세부담 예측**: 현재 추세 기반 연말 예상 세액 자동 계산 + 신뢰구간
- **적립금 가치 추적**: 적립 시점 vs 현재 시점 토큰 가치 비교, 평가손익 표시
- **팀 전체 비용 분석**: 총 인건비 + 적립금 합산 → 월별 실질 인건비 트렌드

### 기술 스택

| 계층 | 기술 |
|------|------|
| Frontend | Next.js + Recharts 또는 Tremor (차트 라이브러리) |
| Backend | FastAPI (기존 Hiring 연동) |
| DB | PostgreSQL (분석 쿼리 최적화) |
| 시각화 | Recharts / Tremor / D3.js |
| 외부 API | 업비트 API, 환율 API |
| 캐싱 | Redis (시세 데이터 캐싱) |
| 배포 | Docker + Vercel/Railway |

### 차별점

- **의사결정 지원**: 단순 기록이 아닌, "급여를 얼마로 하면 세금이 어떻게 되나" 시나리오 분석
- **토큰 가격 연동 인사이트**: 크립토 특유의 변동성을 인건비 관점에서 시각화
- **경영진 관점**: CFO/CEO가 한 눈에 인건비 현황과 전망을 파악

## 컨셉 D: 워크플로우/자동화 중심형 — "TokaFlow"

> **핵심 철학**: 반복 업무는 시스템이 하고, 사람은 승인만 한다.

### UI/UX 구조

**사이드바 메뉴:**
- 📬 인박스 (승인 대기)
- 🔄 워크플로우
- 💰 급여
- 📊 세금
- 👥 팀원
- 🎯 채용
- 📋 로그
- ⚙️ 설정

**네비게이션 흐름:**
인박스가 랜딩. 모든 액션은 "요청 → 승인 → 실행 → 알림" 워크플로우. 급여 지급일이 되면 시스템이 자동으로 급여 내역을 생성하고 관리자에게 승인 요청. 승인하면 기록 확정 + 세금 시뮬레이션 자동 갱신.

**주요 화면:**
- **인박스**: 승인 대기 목록 (급여 확정, 신규 팀원 온보딩, 퇴사 처리 등)
- **워크플로우 설정**: 자동화 규칙 관리 (매월 X일 급여 생성, 분기말 인센티브 생성, 토큰 시세 자동 수집)
- **급여**: 자동 생성된 급여 내역 확인/수정/승인
- **로그**: 모든 자동화 실행 이력, 알림 이력

### 기능 목록

| 영역 | 기능 |
|------|------|
| 급여 | 자동 급여 내역 생성, 승인 워크플로우, 환율/시세 자동 수집 |
| 세금 | 급여 확정 시 자동 재계산, 적립금 자동 산출 |
| 채용 | Hiring 합격 → 온보딩 워크플로우 자동 트리거 |
| 팀원 | 온보딩 체크리스트 자동 생성, 계약 만료 알림 |
| 근태 | 자동 리마인더 (미기록 시 알림) |
| 퇴사 | 퇴사 워크플로우 (최종 정산 → 적립금 정리 → 계정 비활성화) |
| 알림 | Slack/Telegram 연동, 이메일 알림 |
| 스케줄러 | Cron 기반 자동 실행 (시세 수집, 급여 생성, 리마인더) |

### 급여/소득세 계산 로직

A 컨셉과 동일한 기본 로직 + 자동화 레이어:

- **자동 급여 생성**: 매월 지급일 D-3에 급여 초안 자동 생성 → 환율/시세 자동 입력
- **지급일 확정**: 지급일 당일 최종 환율/시세로 자동 업데이트 → 관리자 승인 대기
- **승인 → 확정**: 승인 시 세금 시뮬레이션 자동 재계산, 적립금 자동 산출
- **분기 자동 정산**: 분기말 인센티브 지급 시 연간 누적 소득 자동 재계산 → 적립금 조정분 산출
- **알림**: 세부담 급변 시 (토큰 가격 급등 등) 경고 알림

### 기술 스택

| 계층 | 기술 |
|------|------|
| Frontend | Next.js |
| Backend | FastAPI + Celery (비동기 태스크) |
| DB | PostgreSQL |
| 스케줄러 | Celery Beat 또는 APScheduler |
| 메시징 | Redis (태스크 큐) |
| 알림 | Slack Webhook, Telegram Bot API |
| 외부 API | 업비트 API, 환율 API |
| 배포 | Docker Compose (API + Worker + Redis + DB) |

### 차별점

- **자동화 최우선**: 관리자가 직접 데이터를 입력하는 일이 거의 없음
- **승인 기반 거버넌스**: 모든 중요 액션에 승인 단계 → 실수 방지, 감사 추적
- **이벤트 드리븐**: 채용 합격, 급여일, 분기말 등 이벤트에 반응하는 시스템
- **Slack/Telegram 연동**: 시스템에 들어가지 않아도 알림으로 상황 파악 + 승인

## 컨셉 E: 모듈형 플랫폼 — "TokaOS"

> **핵심 철학**: 필요한 것만 꽂아 쓰는 레고 블록 — 지금은 3개, 나중엔 10개.

### UI/UX 구조

**탑바:**
- 🏠 홈 | 현재 활성 모듈 탭들 (동적)

**사이드바 (모듈별 컨텍스트):**
- 각 모듈 진입 시 해당 모듈의 서브메뉴가 사이드바에 표시
- 예: 급여 모듈 → 급여 기록 / 지급 이력 / 설정

**네비게이션 흐름:**
홈 화면에 설치된 모듈이 카드로 표시 (앱 런처 느낌). 모듈 클릭 → 해당 모듈의 독립 UI로 진입. 모듈 간 데이터는 공유 API 레이어로 연결. 모듈 마켓에서 새 모듈 활성화/비활성화 가능.

**주요 화면:**
- **홈 (앱 런처)**: 설치된 모듈 카드 그리드 — 📦 급여, 📊 세금, 🎯 채용, 👥 팀원, 📅 근태…
- **모듈 내부**: 각 모듈이 독립된 UI/UX 보유 (풀페이지)
- **모듈 마켓**: 사용 가능한 모듈 목록, 활성화 토글
- **통합 설정**: 팀원 정보, API 키, 알림 채널 등 공통 설정

### 기능 목록 (모듈별)

| 모듈 | 상태 | 기능 |
|------|------|------|
| 💰 급여 모듈 | 코어 | USDT/TOKAMAK 급여 기록, 환율/시세 연동, 지급 이력 |
| 📊 세금 모듈 | 코어 | 소득세 시뮬레이션, 적립금 관리, What-if 분석 |
| 👥 팀원 모듈 | 코어 | 프로필, 계약, 지갑 주소 관리 |
| 🎯 채용 모듈 | 통합 | 기존 Hiring 시스템 래핑 (iframe 또는 API 통합) |
| 📅 근태 모듈 | 선택 | 출퇴근 기록, 근무일수 통계 |
| 📝 계약 모듈 | 선택 | 계약서 관리, 갱신 알림 |
| 🚪 오프보딩 모듈 | 선택 | 퇴사 체크리스트, 최종 정산 |
| 📢 알림 모듈 | 선택 | Slack/Telegram 알림 설정 |

### 급여/소득세 계산 로직

A 컨셉과 동일한 기본 로직. 모듈 간 데이터 흐름:

- **급여 모듈 → 세금 모듈**: 급여 확정 이벤트 발행 → 세금 모듈이 구독하여 자동 재계산
- **세금 모듈 → 적립금**: 시뮬레이션 결과 기반 적립금 산출 → 팀원 모듈에 잔고 표시
- **모듈 간 통신**: 이벤트 버스 (내부 pub/sub) 또는 공유 DB 레이어
- **독립 계산 엔진**: 세금 계산 로직은 독립 모듈로 분리 → 세법 변경 시 해당 모듈만 업데이트
- **API 우선**: 모든 계산 결과는 REST API로 노출 → 외부 연동 용이

### 기술 스택

| 계층 | 기술 |
|------|------|
| Frontend | Next.js (Module Federation 또는 동적 라우팅) |
| Backend | FastAPI (모듈별 라우터 분리) |
| DB | PostgreSQL (스키마 기반 모듈 격리) |
| 모듈 통신 | 이벤트 버스 (내부) 또는 공유 서비스 레이어 |
| 외부 API | 업비트 API, 환율 API |
| 배포 | Docker Compose (모놀리스지만 논리적 모듈 분리) |
| 패키지 구조 | 모노레포 (Turborepo) — packages/payroll, packages/tax, packages/hiring… |

### 차별점

- **점진적 확장**: MVP는 급여+세금+팀원 3개 모듈로 시작, 필요 시 추가
- **관심사 분리**: 각 모듈이 독립적이라 개발/유지보수가 깔끔
- **재사용성**: 세금 계산 모듈을 다른 프로젝트에서도 사용 가능
- **미래 대비**: 조직 성장 시 성과 관리, OKR, 비용 관리 등 모듈 추가 가능
- **기존 Hiring 자연스러운 통합**: 별도 모듈로 래핑하여 독립성 유지
