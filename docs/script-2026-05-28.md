Second — net pay wasn't including expenses.

## Slide 7 — New: Org Benchmark Comparison (60 sec)

This is the new idea. When we evaluate a candidate's repo, we now compare it against Tokamak Network's own GitHub repos from the last 6 months.

The system clones recent active repos from our org, measures quality metrics, and builds a benchmark profile. Then when AI analyzes a candidate, it gets this benchmark as context and tells us — is this candidate above, on par, or below our own standards?

One thing we learned early — simple metrics like file count or commit count don't work. Our repos have been maintained for years, so they'll always have more files and commits. That's not a fair comparison.

So we changed to quality-density metrics — things like test presence, documentation structure, code organization, and language alignment. These don't depend on how old the repo is.

This is a first version. Still thinking about better ways to make the comparison more accurate. But it already gives us useful context when reviewing candidates.

---

## Slide 8 — Benchmark Categories (30 sec)

Four categories we're using now:

- Test Presence — does the repo have tests?
- Documentation — is the README structured with sections and install guide?
- Code Organization — config files, CI/CD, modular directory structure
- Language Alignment — how much overlap with our tech stack

Each one shows above, on par, or below compared to our org average.
