## 1. AI Analysis — Tokamak-Specific Evaluation

- Generic evaluation prompt ("Relevance to Ethereum L2")

## 2. Evaluation Criteria — Replaced 2 Items

### Before
| Item | Issue |
|------|-------|
| Innovation | Subjective, no concrete basis from repo |
| AI Proficiency | Cannot determine from repo alone |

### After
| Item | What It Measures |
|------|-----------------|
| **Contribution Potential** | Can this work actually contribute to Tokamak? (extends existing repos, fills ecosystem gap, new tooling) |
| **Deliverable Completeness** | Is there a working, runnable result? (deployed demo, tests, CI/CD) |

### Final 5 Scoring Dimensions
| Dimension | Weight | Basis |
|-----------|--------|-------|
| Technical Completeness | 1.0x | Code quality, architecture, tests |
| Ecosystem Fit | **2.0x** | Tokamak/OP Stack/L2 relevance |
| Tokenomics Impact | 1.0x | TON economics understanding |
| Contribution Potential | 1.0x | Synergy with existing work or new direction |
| Deliverable Completeness | 1.0x | Working demo, deployment, tests |

## 3. Reviewer Matching — Domain-Based

### Before
- Language-only matching (Solidity → Solidity person)
- Two separate systems: "Recommended Reviewers" + "Team Matching" (redundant)
- Candidate description ignored in matching

### After
- **Language + Domain matching** (rollup, bridge, staking, ZK, DeFi, monitor, tokamak, etc.)
- Domain weighted **1.5x** over language
- **Unified into one system** (Recommended Reviewers & Team Matching merged)
- **Description + README + AI Report** all used for keyword extraction
- Tokamak repo mapping: tokamak-thanos → rollup+sequencer+bridge, ton-staking-v2 → staking, etc.
- "Why" explanation: `Domain: rollup, zk | Skills: solidity, typescript`

### Example: Miirza Baig
**Before**: Matched by TypeScript/JavaScript → generic fullstack developers
**After**: Description mentions "tokamak chain monitor", "trh ecosystem", "l2 rollups" → matched to Theo Lee (tokamak-thanos + SentinAI repos, rollup+monitor domain)
