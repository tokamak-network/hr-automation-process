# Transaction Classifier Guide
## 거래 분류기 가이드

---

### 이 기능은 무엇인가요?

법인 은행 거래내역(Aspire, WISE)을 **회계 계정**에 자동으로 분류하는 기능입니다.

예를 들어:
- "Customer A"로부터 입금 → **Sales** (매출)
- "Anthropic"에 출금 → **Subscription fee** (구독료)
- "Member 5"에 출금 → **Salary** (급여)

매년 가결산 시 거래 하나하나를 수동으로 분류하던 작업을 자동화합니다.

---

### 핵심 개념

#### 1. Chart of Accounts (계정 체계)

회사의 모든 수입/지출을 분류하는 카테고리입니다.

| 구분 | 계정 | 설명 |
|------|------|------|
| **수입** | Sales | Customer A 매출 |
| **원가** | Consulting fee | Vendor B 외주비 |
| **비용** | Salary | 직원 급여 |
| | Director remuneration | 이사 보수 |
| | Subscription fee | SaaS 구독 (Anthropic, GitHub 등) |
| | Professional fee | 법무/세무 자문 |
| | Office rental | 사무실 임대 |
| | Secretary fee | 법인 Secretary |
| | Travel expense | 출장/워크샵 |
| | Bank charges | 은행 수수료 |
| **기타수입** | Other income | 캐시백, 이자 등 |
| **기타비용** | Exchange Gain or Loss | 환차손익 |
| **내부** | Internal transfer | 계좌간 이체 |

#### 2. Counterparty Rules (분류 룰)

**거래상대 이름** → **계정** 매핑 규칙입니다.

```
"Anthropic" → Subscription fee
"OpenAI"    → Subscription fee
"Vendor C" → Professional fee
```

한 번 등록하면 같은 거래상대의 모든 거래가 자동 분류됩니다.

#### 3. WHT Flag (원천세 플래그)

한국 거주자에게 송금 시 원천세 검토가 필요할 수 있습니다.
룰에 WHT 플래그가 설정된 거래는 빨간색으로 경고 표시됩니다.

해당 거래: Vendor B, Vendor C, Vendor D 등

---

### 사용 방법

#### Step 1: 일괄 분류 실행

1. **"일괄 분류 실행"** 버튼 클릭
2. 등록된 룰에 매칭되는 거래가 자동 분류됩니다
3. 요약 카드에서 분류 현황을 확인합니다

#### Step 2: 미분류 거래 처리

자동 분류가 안 된 거래는 **미분류** 목록에 표시됩니다.

1. 미분류 거래의 **드롭다운**에서 계정을 선택
2. 자동으로 해당 거래상대가 룰로 저장됩니다
3. 다음부터 같은 거래상대는 자동 분류

#### Step 3: 룰 관리

**"룰 관리"** 버튼을 클릭하면 등록된 모든 룰을 확인할 수 있습니다.

- 패턴: 거래상대 이름에 포함된 텍스트 (대소문자 무시)
- 계정: 매핑될 회계 계정
- 거주지: 거래상대의 세금 거주국
- WHT: 원천세 검토 필요 여부

#### Step 4: FY(회계연도) 필터

회계연도별로 분류 현황을 확인할 수 있습니다.

- FY24: 2023년 3월 ~ 2024년 2월
- FY25: 2024년 3월 ~ 2025년 2월
- FY26: 2025년 3월 ~ 2026년 2월

---

### 분류 정확도

| 시점 | 예상 자동 분류율 | 소요 시간 |
|------|----------------|----------|
| 최초 실행 | ~70% | 미분류 수동 처리 1-2시간 |
| 2번째 FY | ~90% | 신규 거래상대만 처리 20분 |
| 3번째 FY 이후 | ~95%+ | 5분 |

---

### 계정별 요약 테이블

일괄 분류 후 계정별로 입금/출금 합계를 확인할 수 있습니다.
이 데이터가 가결산 시 PL(손익계산서)의 기초 자료가 됩니다.

---

### FAQ

**Q: 분류를 잘못했으면?**
A: 법인 입출금 탭에서 해당 거래를 찾아 account_code를 수정하거나, 룰 관리에서 룰을 수정/삭제할 수 있습니다.

**Q: 같은 거래상대가 다른 계정에 해당하면?**
A: 현재는 하나의 거래상대에 하나의 계정만 매핑됩니다. 예외적인 거래는 수동으로 분류해주세요.

**Q: 내부 이체(Aspire → WISE)도 분류되나?**
A: "Internal transfer"로 분류됩니다. 회계상 자산 이동이므로 PL에는 영향 없습니다.

**Q: WHT 플래그는 실제 세금 계산인가?**
A: 아닙니다. 한국 거주자 송금에 대한 **검토 알림**입니다. 실제 원천세 계산은 세무사와 확인이 필요합니다.
